Alpha tecnica

Voz moderna para Python sin depender de PyAudio como base.

AuralisVoiceKit define un core liviano para captura, transcripcion, eventos y salida de voz. English: a modern voice toolkit for Python assistants, local agents and voice automation tools. La instalacion base no obliga a compilar extensiones ni a esperar wheels de una version especifica de CPython. MP3 y FLAC funcionan mediante ffmpeg como herramienta externa opcional.

CapturaMicrofono, archivo o backend simulado.
EventosEstado observable sin filtrar audio por defecto.
TranscripcionLocal, API o adaptador personalizado.
SalidaVoz sintetica o eventos para asistentes.

Instalacion

Durante el desarrollo local se instala en modo editable.

cd E:\AuralisVoiceKit
py -m pip install -e .

El core no trae dependencias externas. Los backends reales se instalan como extras opcionales.

py -m pip install -e ".[sounddevice]"
py -m pip install -e ".[openai]"
py -m pip install -e ".[whisper]"

Desde un repositorio clonado, el ejemplo pequeno de integracion para usuarios de PyPI funciona sin microfono, modelos ni credenciales.

python -m pip install auralisvoicekit
python examples/pypi_quickstart.py --json

Inicio rapido

El backend por defecto es null, util para probar integraciones sin hardware.

from auralis_voicekit import AuralisVoiceKit, AudioChunk, AudioFormat

kit = AuralisVoiceKit()

chunk = AudioChunk(
    data=b"\x00\x00" * 16000,
    format=AudioFormat(sample_rate=16000, channels=1, sample_width=2),
)

result = kit.transcribe(chunk)
print(result.text)

El archivo examples/pypi_quickstart.py genera audio sintetico, escribe un WAV, segmenta voz y transcribe con el backend null. Es la prueba recomendada para validar el paquete base desde un repositorio clonado.

python examples/pypi_quickstart.py --json

Captura real

Con el extra sounddevice, se puede listar dispositivos y grabar a WAV desde el ejemplo incluido.

py -m pip install -e ".[sounddevice]"
py -m auralis_voicekit.cli devices --backend sounddevice
py -m auralis_voicekit.cli devices --backend wasapi
py -m auralis_voicekit.cli doctor --devices --backend wasapi --json
py examples\capture_microphone.py --seconds 3 --output capture.wav

En Windows, wasapi filtra dispositivos por la host API WASAPI usando el extra opcional sounddevice. El diagnostico WASAPI muestra host APIs, ids WASAPI, dispositivo default y el dispositivo que se usaria al pedir default.

from auralis_voicekit import AuralisVoiceKit, VoiceKitConfig
from auralis_voicekit.backends import inspect_wasapi_environment

snapshot = inspect_wasapi_environment()
print(snapshot.to_dict())

kit = AuralisVoiceKit(VoiceKitConfig(capture_backend="wasapi"))
kit.start_capture()

Segmentacion de voz

El core puede calibrar ruido y agrupar chunks en segmentos de voz sin instalar modelos ni NumPy.

from auralis_voicekit import VoiceActivityDetector, calibrate_noise_pcm16, normalize_pcm16

profile = calibrate_noise_pcm16(noise_chunks)
detector = VoiceActivityDetector(noise_profile=profile)
segments = detector.segment(recorded_chunks)
normalized = normalize_pcm16(recorded_chunks[0], target_peak=0.95)
py examples\capture_voice_segments.py --calibrate-seconds 1 --record-seconds 5

Normalizacion

La normalizacion PCM16 ajusta volumen con proteccion de clipping y puede generar un WAV listo para inspeccionar o transcribir.

auralis normalize sample.mp3 normalized.wav --target-peak 0.95
auralis normalize sample.flac normalized.wav --target-peak 0.95
py examples\normalize_audio.py sample.mp3 normalized.wav

WAV offline

Los archivos WAV PCM16 pueden inspeccionarse, leerse por chunks y usarse como backend de captura.

py -m auralis_voicekit.cli wav-info sample.wav
py examples\segment_wav.py sample.wav --output-dir wav_segments
from auralis_voicekit import AuralisVoiceKit, VoiceKitConfig

kit = AuralisVoiceKit(
    VoiceKitConfig(capture_backend="wav", input_file="sample.wav")
)

Transcripcion

El backend por defecto es null. Las rutas reales de transcripcion son explicitas: whisper para transcripcion local y openai para transcripcion por API. El paquete base sigue sin dependencias externas; WAV se lee directo y MP3/FLAC se decodifican con ffmpeg cuando esta disponible.

Whisper local

El backend whisper usa faster-whisper. No requiere API key, pero puede descargar modelos en el primer uso y agrega dependencias de ML fuera del core.

py -m pip install -e ".[whisper]"
auralis transcribe sample.wav --backend whisper --model base --language es
auralis transcribe sample.mp3 --backend whisper --model base --normalize --json
auralis transcribe-segments sample.mp3 --backend whisper --model small --normalize --json
from auralis_voicekit import AuralisVoiceKit, VoiceKitConfig, read_audio_as_chunk

chunk = read_audio_as_chunk("sample.mp3")
kit = AuralisVoiceKit(
    VoiceKitConfig(
        transcription_backend="whisper",
        transcription_model="base",
        transcription_device="auto",
        transcription_compute_type="default",
        transcription_beam_size=5,
        transcription_vad_filter=False,
        language="es",
    )
)

result = kit.transcribe(chunk)
print(result.text)

OpenAI por API

El backend opcional openai usa el cliente oficial de OpenAI. Requiere OPENAI_API_KEY y no agrega dependencias nativas de audio al paquete base.

py -m pip install -e ".[openai]"
$env:OPENAI_API_KEY="tu_api_key"
auralis transcribe sample.wav --backend openai --language es
auralis transcribe sample.mp3 --backend openai --language es
auralis transcribe sample.flac --backend openai --language es
auralis transcribe sample.mp3 --backend openai --normalize --target-peak 0.95
auralis transcribe sample.wav --backend openai --model gpt-4o-transcribe --json
py examples\transcribe_wav.py sample.mp3 --backend openai

Desde Python, se lee un archivo de audio como AudioChunk y se llama a kit.transcribe().

from auralis_voicekit import AuralisVoiceKit, VoiceKitConfig, read_audio_as_chunk

chunk = read_audio_as_chunk("sample.mp3")
kit = AuralisVoiceKit(
    VoiceKitConfig(
        transcription_backend="openai",
        transcription_model="gpt-4o-mini-transcribe",
        language="es",
    )
)

result = kit.transcribe(chunk)
print(result.text)

Ambos backends convierten el audio a un flujo compatible y devuelven un TranscriptResult con texto, idioma, fuente y metadatos. Referencias: faster-whisper y OpenAI speech-to-text.

Diagnostico de ffmpeg

Para MP3, FLAC y otros formatos comprimidos, la libreria usa ffmpeg como herramienta externa opcional. Si falta o falla, el error incluye rutas revisadas, sugerencia de instalacion por sistema operativo, comando usado, stderr truncado y una prueba manual con ffmpeg -hide_banner -i.

$env:AURALIS_FFMPEG_PATH="C:\Tools\ffmpeg\bin\ffmpeg.exe"
auralis transcribe sample.mp3 --backend null --ffmpeg "C:\Tools\ffmpeg\bin\ffmpeg.exe"
auralis doctor --json
from auralis_voicekit import ffmpeg_install_hint, ffmpeg_search_locations

print(ffmpeg_install_hint())
print(ffmpeg_search_locations())

Sesiones

VoiceSession ofrece un flujo de alto nivel para leer o capturar audio, segmentar voz y transcribir cada turno. Tambien puede cancelarse desde otro hilo, desde callbacks o al salir de un contexto.

from auralis_voicekit import AuralisVoiceKit, VoiceKitConfig, VoiceSession

kit = AuralisVoiceKit(
    VoiceKitConfig(
        transcription_backend="whisper",
        transcription_model="base",
        language="es",
    )
)

with VoiceSession(kit) as session:
    for turn in session.transcribe_file("sample.mp3"):
        print(turn.index, turn.text)
with VoiceSession(kit) as session:
    turns = session.listen_once(
        30.0,
        on_turn=lambda turn: False,
    )

La CLI tambien puede segmentar y transcribir audio por turnos.

auralis transcribe-segments sample.wav --backend whisper --model base --language es --json
auralis transcribe-segments sample.wav --backend openai --language es --json
auralis transcribe-segments sample.mp3 --backend openai --language es --json
auralis transcribe-segments sample.mp3 --backend openai --normalize --json
auralis transcribe-segments sample.wav --backend null --json
py examples\assistant_loop.py --file sample.mp3 --transcription-backend whisper --model base
py examples\assistant_loop.py --seconds 30 --capture-backend sounddevice --transcription-backend whisper --model base
py examples\local_assistant_privacy_demo.py --json

Salida de voz

El backend system permite hablar con herramientas del sistema operativo. Windows usa PowerShell/SAPI, macOS usa say y Ubuntu/Linux usa spd-say o espeak cuando estan disponibles. Tambien puede listar voces y aplicar voz, velocidad o volumen cuando la plataforma lo soporta.

auralis speak "Hola desde AuralisVoiceKit" --backend null --json
auralis speak "Hola desde AuralisVoiceKit" --backend system
auralis voices --backend system
auralis speak "Hola desde AuralisVoiceKit" --backend system --voice "Microsoft Helena" --rate 2 --volume 80
python examples/system_output_demo.py --json
python examples/system_output_demo.py --system Windows --voice "Microsoft Helena" --rate 2 --volume 80 --json
kit = AuralisVoiceKit(
    VoiceKitConfig(
        output_backend="system",
        output_voice="Microsoft Helena",
        output_rate=2,
        output_volume=80,
    )
)
kit.speak("Hola desde AuralisVoiceKit")

English: the system output backend supports voice listing and voice/rate/volume selection where the operating system command exposes those controls.

examples/system_output_demo.py usa dry-run por defecto para registrar el comando que se ejecutaria y emitir eventos sin reproducir audio. tools/output_pilot.py genera artifacts JSON y Markdown para pilotos de salida, con el texto redactado dentro de comandos. Para un piloto real que pueda cerrar el blocker beta se debe pasar --speak --operator-present --confirm-audible de forma explicita.

python examples/system_output_demo.py --speak --text "Hola desde AuralisVoiceKit"
python tools/output_pilot.py --output-dir pilot_runs/output/system-dry-run --json
python tools/output_pilot.py --speak --operator-present --confirm-audible --text "Hola desde AuralisVoiceKit" --json

Diagnostico

auralis doctor produce checks de Python, plataforma, dependencias opcionales, backends, dispositivos, archivos WAV y busqueda de ffmpeg. Para wasapi, el reporte incluye un snapshot de host APIs y dispositivos de entrada. MP3 y FLAC se validan con pruebas de integracion reales cuando ffmpeg esta disponible. Los pilotos pueden generar bundles sanitizados que redactan rutas locales y nombres de dispositivos, y luego analizarlos por sistema, check, categoria y prioridad.

auralis doctor
auralis doctor --devices --backend wav
auralis doctor --devices --backend sounddevice
auralis doctor --devices --backend wasapi --json
auralis doctor --capture-test --backend sounddevice --capture-seconds 0.25
auralis doctor --capture-test --backend wasapi --device default --sample-rate 48000 --json
auralis doctor --capture-test --backend sounddevice --device default --json
auralis doctor --wav sample.wav
auralis doctor --json
auralis doctor --devices --backend wasapi --bundle reports/doctor-windows.json
auralis doctor-bundles reports/doctor-windows.json --output reports/doctor-analysis.json --json

Los warnings no bloquean el core. Por ejemplo, sounddevice puede faltar y aun asi funcionar el backend wav. El flag --capture-test intenta abrir brevemente el backend seleccionado para diagnosticar permisos o errores de dispositivo.

El flag --bundle escribe un JSON seguro para soporte: no recoge audio ni transcripciones y marca que rutas locales y nombres de dispositivos fueron redactados. doctor-bundles ayuda a comparar varios reportes y ordenar los hallazgos.

En Windows, los fallos de captura incluyen windows_audio_hint con acciones para permisos de microfono, dispositivo invalido, sample rate, canales y errores de host API.

from auralis_voicekit import windows_audio_error_hint

hint = windows_audio_error_hint(
    "Unanticipated host error [PaErrorCode -9999]",
    backend="wasapi",
    system="Windows",
)
print(hint.format_hint())
from auralis_voicekit import analyze_doctor_bundles, run_doctor, write_doctor_bundle

report = run_doctor(include_devices=True, capture_backend="wasapi")
write_doctor_bundle("reports/doctor-windows.json", report)
analysis = analyze_doctor_bundles(["reports/doctor-windows.json"])
print(analysis.priority_counts)

Piloto seguro

tools/pilot_run.py ejecuta un piloto automatizado sin microfono, sin audio real, sin red y sin modelos. Tambien puede recibir artifacts JSON con --evidence, devolver next_beta_evidence_steps y escribir pilot-plan.md con evidencias aceptadas/ignoradas, recommended_pilot_sequence, comandos reales pendientes y campos requeridos. tools/manual_pilot.py genera bundle doctor, analisis doctor-bundles, reporte JSON y Markdown de hallazgos; no abre microfono salvo que se use --capture-test. tools/output_pilot.py prepara pilotos de salida system sin reproducir audio por defecto. tools/transcription_pilot.py prepara pilotos de transcripcion con audio sintetico y backend null por defecto, y puede calcular scoring redactado con --expected-text o --expected-text-file. tools/beta_readiness.py genera BETA_CHECKLIST.md con blockers de beta, acepta artifacts JSON reales mediante --evidence, imprime requisitos de evidencia con --requirements, audita artifacts y resume blockers cerrados/pendientes con --audit-evidence, puede fallar con --fail-on-audit-gaps, exige project: AuralisVoiceKit y reporta evidencias ignoradas con motivos seguros como missing_project, wrong_project o not_json_object. English: safe pilot plans and strict evidence audits avoid exposing local paths or private content while keeping an ordered real-pilot sequence. Los pasos manuales viven en PILOTS.md y los hallazgos resumidos en PILOT_FINDINGS.md.

Benchmarks de latencia

auralis benchmark mide una linea base offline y determinista sin microfono, red ni extras nativos. Genera audio PCM16 sintetico, lo captura desde el backend wav, segmenta por RMS y transcribe los segmentos con el backend elegido. auralis benchmark-whisper compara configuraciones locales de faster-whisper cuando el extra whisper esta instalado. Ambos comandos pueden exportar reportes JSON/CSV para CI, pilotos reales o comparaciones de hardware.

auralis benchmark
auralis benchmark --iterations 10 --duration 3 --json
auralis benchmark --iterations 10 --output reports/offline.json
auralis benchmark --iterations 10 --output reports/offline.csv --output-format csv
auralis benchmark --transcription-backend whisper --model base --iterations 3
auralis benchmark-whisper --models tiny,base --beam-sizes 1,5 --iterations 3 --json
auralis benchmark-whisper --models tiny,base --beam-sizes 1,5 --output reports/whisper.csv

Por defecto se usa transcription:null; eso mide el costo del pipeline, no un modelo real. Para medir transcripcion local real instala .[whisper] y usa --transcription-backend whisper o benchmark-whisper.

from auralis_voicekit import (
    run_offline_benchmarks,
    run_whisper_comparison_benchmarks,
    write_benchmark_report,
)

report = run_offline_benchmarks(iterations=5)
for result in report.results:
    print(result.name, result.mean_ms, result.p95_ms)
write_benchmark_report(report, "reports/offline.csv")

comparison = run_whisper_comparison_benchmarks(models=("tiny", "base"), beam_sizes=(1, 5))
print(comparison.to_dict()["rankings"])
write_benchmark_report(comparison, "reports/whisper.json")

Arquitectura

La libreria se organiza en contratos pequenos y backends intercambiables.

src/auralis_voicekit/
  config.py          VoiceKitConfig
  diagnostics.py     Reporte doctor estructurado
  models.py          AudioChunk, AudioFormat, TranscriptResult
  events.py          EventBus, VoiceEventType
  audio.py           RMS, pico, silencio, normalizacion, segmentos y WAV PCM16
  benchmarks.py      Benchmarks offline y comparativos para Whisper
  kit.py             AuralisVoiceKit
  session.py         VoiceSession, VoiceSessionConfig, VoiceTurn
  ffmpeg             Decodificacion opcional de MP3/FLAC a PCM16 con errores accionables
  backends/
    base.py          Protocolos publicos
    null.py          Backend seguro para pruebas
    wav_file.py      Backend offline WAV PCM16
    sounddevice.py   Backend opcional funcional
    wasapi.py         Backend inicial Windows WASAPI con diagnostico
    whisper_transcription.py  Backend opcional de transcripcion local
    openai_transcription.py  Backend opcional de transcripcion por API
    system_output.py Backend opcional de salida de voz del sistema
    registry.py      Registro de factories

API publica

AuralisVoiceKit

Fachada principal. Compone configuracion, eventos y backends.

kit = AuralisVoiceKit()
kit.start_capture()
kit.stop_capture()
result = kit.transcribe(chunk)
kit.speak("Hola")

VoiceKitConfig

Controla formato de audio, idioma, privacidad y seleccion de backends.

from auralis_voicekit import VoiceKitConfig

config = VoiceKitConfig(
    capture_backend="sounddevice",
    transcription_backend="null",
    sample_rate=16000,
    channels=1,
    privacy_mode=True,
)

EventBus

Permite observar el flujo de voz sin acoplar el asistente al backend concreto.

from auralis_voicekit import VoiceEventType

kit.events.subscribe(
    VoiceEventType.TRANSCRIPTION_COMPLETED,
    lambda event: print(event.payload),
)

VoiceSession

Une captura o lectura de archivos de audio con segmentacion y transcripcion por turnos. Expone cancelacion y cierre ordenado para loops largos.

with VoiceSession(kit) as session:
    turns = session.transcribe_file("sample.mp3")
    live_turns = session.listen_once(5.0)
    session.cancel()

Backends

Backend Estado Uso
null Incluido Pruebas, demos y desarrollo sin microfono.
wav Inicial funcional Captura offline desde archivos WAV PCM16 para pruebas y demos.
sounddevice Inicial funcional Captura moderna con dependencia opcional, seleccion de dispositivo y cierre robusto.
wasapi Inicial con diagnostico reforzado Captura Windows filtrada por host API WASAPI y snapshot de host APIs/dispositivos.
whisper Inicial funcional Transcripcion local opcional con faster-whisper.
openai Inicial funcional Transcripcion por API.
system Inicial con voces configurables Salida de voz con herramientas del sistema operativo, listado de voces y parametros de voz.

Privacidad y logs

Con privacy_mode=True, los eventos no incluyen texto transcrito ni audio crudo. El asistente puede observar duraciones, backends y estados sin registrar datos sensibles por accidente.

PrivacyEventLogger permite exportar eventos como JSON Lines con payload sanitizado. Campos como text, prompt, token, path, audio y raw se reemplazan por [redacted].

from auralis_voicekit import AuralisVoiceKit, PrivacyEventLogger

kit = AuralisVoiceKit()

with PrivacyEventLogger("auralis-events.jsonl") as logger:
    unsubscribe = logger.subscribe(kit.events)
    result = kit.transcribe(chunk)
    unsubscribe()
