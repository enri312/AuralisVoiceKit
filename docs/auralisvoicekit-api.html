Instalacion

El paquete base no instala dependencias nativas. Esa decision mantiene el core compatible con Python moderno y con versiones futuras razonables.

python -m pip install auralisvoicekit

Extras opcionales:

python -m pip install "auralisvoicekit[sounddevice]"
python -m pip install "auralisvoicekit[openai]"
python -m pip install "auralisvoicekit[whisper]"

Durante desarrollo local:

python -m pip install -e .
python -m auralis_voicekit.cli doctor
python -m unittest discover -s tests

Inicio rapido

El backend por defecto es null. Permite verificar integraciones sin microfono, red, modelos ni credenciales.

from auralis_voicekit import AuralisVoiceKit, AudioChunk, AudioFormat

kit = AuralisVoiceKit()
chunk = AudioChunk(
    data=b"\x00\x00" * 16000,
    format=AudioFormat(sample_rate=16000, channels=1, sample_width=2),
)

result = kit.transcribe(chunk)
print(result.text)

Modelos

Simbolo Uso Campos principales
AudioEncoding Enum de codificacion de audio. PCM16, FLOAT32
AudioFormat Formato de un chunk de audio. sample_rate, channels, sample_width, encoding, bytes_per_second
AudioChunk Bloque de audio con metadata. data, format, timestamp, metadata, duration_seconds
TranscriptResult Resultado de transcripcion. text, confidence, language, is_final, source, metadata
AudioDevice Dispositivo de entrada reportado por un backend. id, name, kind, channels, host_api, is_default, metadata

Configuracion

VoiceKitConfig concentra formato de audio, idioma, privacidad y seleccion de backends.

from auralis_voicekit import VoiceKitConfig

config = VoiceKitConfig(
    sample_rate=16000,
    channels=1,
    language="es",
    capture_backend="wav",
    transcription_backend="null",
    output_backend="system",
    output_voice="Microsoft Helena",
    output_rate=2,
    output_volume=80,
    privacy_mode=True,
)

AuralisVoiceKit

AuralisVoiceKit es la fachada principal. Compone configuracion, registro de backends y bus de eventos.

Metodo Descripcion
start_capture(on_audio=None) Abre el backend de captura y emite eventos capture.started y audio.chunk.
stop_capture() Detiene captura y emite capture.stopped.
transcribe(chunk) Transcribe un AudioChunk con el backend seleccionado.
speak(text) Envia texto al backend de salida. Con output_backend="system" puede usar voz, velocidad y volumen.
backend_report() Devuelve informacion de backends registrados.

Audio comprimido

MP3, FLAC y otros formatos se decodifican con ffmpeg externo.

from auralis_voicekit import (
    decode_audio_with_ffmpeg,
    ffmpeg_available,
    ffmpeg_install_hint,
    ffmpeg_search_locations,
    read_audio,
    read_audio_as_chunk,
    resolve_ffmpeg_executable,
)

print(ffmpeg_available())
print(ffmpeg_search_locations())
chunk = read_audio_as_chunk("sample.mp3")

chunk_audio divide un AudioChunk PCM16 en bloques mas pequenos. chunk_to_wav_bytes codifica un chunk PCM16 como WAV en memoria.

Sesiones

VoiceSession agrupa lectura o captura, segmentacion y transcripcion por turnos.

from auralis_voicekit import AuralisVoiceKit, VoiceKitConfig, VoiceSession, VoiceSessionConfig

kit = AuralisVoiceKit(
    VoiceKitConfig(transcription_backend="whisper", transcription_model="base")
)

with VoiceSession(kit, VoiceSessionConfig(chunk_duration_ms=50)) as session:
    turns = session.transcribe_file("sample.mp3")
    for turn in turns:
        print(turn.index, turn.text)
Simbolo Uso
VoiceSessionConfig Controla chunking, VAD, normalizacion, limite de turnos y ejecutable ffmpeg.
VoiceTurn Une un VoiceSegment y un TranscriptResult. Expone text, duration_seconds, rms y to_dict().
VoiceSession Expone segment_chunks, transcribe_segments, transcribe_chunks, transcribe_wav, transcribe_file, capture_for, listen_once, cancel y close.

Eventos y privacidad

EventBus permite observar el flujo sin acoplarse a un backend. Con privacy_mode=True, los eventos evitan audio crudo y texto transcrito.

from auralis_voicekit import EventBus, VoiceEvent, VoiceEventType

events = EventBus()
unsubscribe = events.subscribe(
    VoiceEventType.ANY,
    lambda event: print(event.type, event.payload),
)
event = events.emit(VoiceEventType.ERROR, {"message": "demo"})
unsubscribe()

Diagnostico

run_doctor() genera un DoctorReport con checks estructurados. La CLI usa la misma API.

from auralis_voicekit import DiagnosticCheck, DiagnosticStatus, DoctorReport, run_doctor

report = run_doctor(include_devices=True, capture_backend="wav")
print(report.status)
print(report.to_dict())

Benchmarks

run_offline_benchmarks() mide una linea base determinista para captura WAV, segmentacion RMS y transcripcion con el backend elegido. run_whisper_comparison_benchmarks() compara varias configuraciones locales de faster-whisper y devuelve un ranking por latencia media.

from auralis_voicekit import (
    BenchmarkComparisonEntry,
    BenchmarkComparisonReport,
    BenchmarkReport,
    BenchmarkResult,
    generate_synthetic_audio_chunks,
    run_offline_benchmarks,
    run_whisper_comparison_benchmarks,
)

report = run_offline_benchmarks(iterations=5)
for result in report.results:
    print(result.name, result.mean_ms, result.p95_ms)

comparison: BenchmarkComparisonReport = run_whisper_comparison_benchmarks(
    models=("tiny", "base"),
    beam_sizes=(1, 5),
    iterations=3,
)
for entry in comparison.entries:
    print(entry.name, entry.transcription_mean_ms)

Errores

Clase Cuando aparece
AuralisError Base comun para errores propios de la libreria.
AudioSourceError Archivos WAV invalidos, ffmpeg faltante o audio comprimido que no puede decodificarse.
AudioDeviceNotFound Selector de dispositivo inexistente o no compatible.
BackendNotAvailable Extra opcional ausente o backend no disponible en el sistema operativo.
PermissionRequired Permisos de microfono u otra autorizacion del sistema.
TranscriptionError Fallo al llamar a un backend de transcripcion real.

Backends personalizados

Los contratos viven en auralis_voicekit.backends.base. Un backend custom implementa los metodos requeridos y se registra en un BackendRegistry.

Diagnostico WASAPI

El backend Windows wasapi expone inspect_wasapi_environment() y WasapiDiagnosticSnapshot desde auralis_voicekit.backends. El snapshot inspecciona host APIs y dispositivos sin abrir un stream de microfono.

from auralis_voicekit.backends import WasapiDiagnosticSnapshot, inspect_wasapi_environment

snapshot: WasapiDiagnosticSnapshot = inspect_wasapi_environment()
print(snapshot.to_dict())
auralis doctor --devices --backend wasapi --json
auralis doctor --capture-test --backend wasapi --device default --json

Salida system

El backend system vive en auralis_voicekit.backends. Puede listar voces del sistema como SystemVoice y aplicar parametros de salida segun plataforma.

from auralis_voicekit import AuralisVoiceKit, VoiceKitConfig
from auralis_voicekit.backends import SystemSpeechOutputBackend, SystemVoice

backend = SystemSpeechOutputBackend()
voices: tuple[SystemVoice, ...] = backend.list_voices()

kit = AuralisVoiceKit(
    VoiceKitConfig(
        output_backend="system",
        output_voice=voices[0].id if voices else None,
        output_rate=2,
        output_volume=80,
    )
)
kit.speak("Hola desde AuralisVoiceKit")
auralis voices --backend system --json
auralis speak "Hola" --backend system --voice es --rate 160 --volume 80

Registro custom

from auralis_voicekit import AuralisVoiceKit, TranscriptResult, VoiceKitConfig
from auralis_voicekit.backends import BackendRegistry, create_default_registry

class MyTranscriber:
    name = "my-transcriber"

    def info(self):
        from auralis_voicekit.backends.base import BackendInfo
        return BackendInfo(name=self.name, kind="transcription")

    def transcribe(self, chunk, config):
        return TranscriptResult(text="hola", language=config.language, source=self.name)

registry = create_default_registry()
registry.register_transcription("my-transcriber", MyTranscriber)
kit = AuralisVoiceKit(
    VoiceKitConfig(transcription_backend="my-transcriber"),
    registry=registry,
)

Contratos disponibles: CaptureBackend, TranscriptionBackend, SpeechOutputBackend y BackendInfo.
